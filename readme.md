# SemantiCheck

> **Advanced Semantic Analysis & AI Content Detection Engine**

A powerful web-based tool that combines semantic plagiarism detection and AI-generated content identification using state-of-the-art machine learning models.

## ğŸš€ Features

### 1. **Semantic Plagiarism Detection**
- Compare two texts (Text A vs Text B) for semantic similarity
- Detects paraphrased content and idea overlap
- Uses Sentence Transformers for high-speed embedding generation
- Provides detailed risk assessment (Low, Medium, High)

### 2. **AI Content Detection**
- Identify whether text was generated by AI (ChatGPT, Claude, etc.) or written by humans
- Analyzes linguistic patterns, structure, and vocabulary
- Powered by Google Gemini 2.5 Flash
- Returns probability score and detailed reasoning in Indonesian

### 3. **File Upload Support**
- Upload `.txt` and `.docx` files for analysis
- Drag-and-drop interface
- Automatic text extraction from Word documents

### 4. **Dual Analysis Modes**
- **Quick Check (Local)**: Fast local analysis using Sentence Transformers
- **Deep Analysis (AI)**: Comprehensive analysis with LLM-powered insights

## ğŸ“¦ Installation

### Prerequisites
- Python 3.13 or higher
- pip package manager
- Google Gemini API Key ([Get one here](https://aistudio.google.com/app/apikey))

### Setup Instructions

1. **Clone the repository**
   ```bash
   git clone https://github.com/FAJRIAG/SemantiCheck.git
   cd SemantiCheck
   ```

2. **Create and activate virtual environment**
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Configure Environment Variables**
   
   Create a `.env` file in the project root:
   ```env
   GEMINI_API_KEY=your_api_key_here
   ```

5. **Run the application**
   ```bash
   python -m uvicorn app.main:app --reload
   ```

6. **Access the application**
   
   Open your browser and navigate to: `http://localhost:8000`

## ğŸ“– Usage Guide

### Plagiarism Detection (Text A vs Text B)

1. Select **"Plagiarism Check (A vs B)"** mode
2. Paste the original text in **Text A**
3. Paste the suspect text in **Text B**
4. Click **"Quick Check (Local)"** for fast analysis or **"Deep Analysis (AI)"** for detailed insights

### AI Content Detection

1. Select **"AI Content Detector"** mode
2. Either:
   - Paste text directly into the input box
   - Upload a `.txt` or `.docx` file
3. Click **"Detect AI Content"**
4. Review the AI probability score, verdict, and reasoning

## ğŸ”Œ API Documentation

### Endpoints

#### 1. Local Similarity Analysis
```http
POST /analyze/local
Content-Type: application/json

{
  "text_a": "Original text here",
  "text_b": "Suspect text here"
}
```

**Response:**
```json
{
  "similarity_score": 0.87,
  "risk_level": "High",
  "message": "Analysis complete"
}
```

#### 2. Detailed Analysis with LLM
```http
POST /analyze/detailed
Content-Type: application/json

{
  "text_a": "Original text here",
  "text_b": "Suspect text here"
}
```

**Response:**
```json
{
  "similarity_score": 0.87,
  "risk_level": "High",
  "detailed_analysis": "Markdown-formatted detailed explanation..."
}
```

#### 3. AI Content Detection (Text)
```http
POST /detect-ai
Content-Type: application/json

{
  "text": "Text to analyze for AI generation"
}
```

**Response:**
```json
{
  "ai_probability": 85,
  "verdict": "Likely AI",
  "reasoning": "Teks menunjukkan pola bahasa yang sangat terstruktur..."
}
```

#### 4. AI Content Detection (File)
```http
POST /detect-ai/file
Content-Type: multipart/form-data

file: [uploaded .txt or .docx file]
```

**Response:** Same as `/detect-ai`

## âš™ï¸ Configuration

### Risk Level Thresholds

Default similarity score thresholds (configurable in `app/core/similarity.py`):

- **Low Risk**: < 40%
- **Medium Risk**: 40% - 70%
- **High Risk**: > 70%

### Model Configuration

- **Embedding Model**: `all-MiniLM-L6-v2` (90MB, downloads on first run)
- **LLM Model**: `gemini-2.5-flash` (requires API key)

## ğŸš¨ Important Notes

### Security
- Never commit your `.env` file to version control
- The `.gitignore` file already excludes `.env` and `.venv`
- Keep your Gemini API key confidential

### Performance
- First startup takes 2-5 minutes (model download)
- Subsequent startups are instant
- Quick Check: ~1-2 seconds per analysis
- Deep Analysis: ~5-10 seconds (depends on LLM response time)

### Limitations
- Supported file formats: `.txt`, `.docx` only
- Maximum text length: Limited by available memory
- AI detection works best with English and Indonesian text
